# Expectations

- Preserve ordered numbering, bullet nesting, task states, indentation depth, and paragraph after the list.
- Normalized output may renumber ordered lists consistently only if semantic ordering remains clear.
- Opaque handling is not expected for supported GFM list syntax; unsupported nested tasks must remain raw.
- Source-only handling is acceptable in rich mode if complex nesting cannot be edited safely.
- Render as ordered, unordered, and task-list structures with nesting intact.
